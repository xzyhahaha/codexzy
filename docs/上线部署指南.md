# codexzy 上线部署指南

这份文档按“服务器上没有 MySQL、没有 JDK 17、甚至没有 Maven”的情况来写，适合你拿一台全新的 Linux 服务器，照着一步一步把项目跑起来。

本文默认采用下面这套方案：

- 应用：Spring Boot Jar 包直接运行
- 数据库：用 Docker 启动 MySQL 8
- JDK：服务器本机安装 OpenJDK 17
- 部署目录：`/opt/codexzy`
- 上传目录：`/data/codexzy/uploads`
- MySQL 数据目录：`/data/codexzy/mysql`

如果你的服务器是 Ubuntu 20.04 / 22.04 / 24.04，这份文档基本可以直接照抄。  
如果是 CentOS 7.6，请优先看下面的“CentOS 7.6 专用说明”。  
如果是 Rocky / AlmaLinux / CentOS Stream 这类新一点的系统，再看文档里 `dnf` 那部分。

## 1. 项目运行要求

当前项目运行需要：

- JDK 17 或更高
- MySQL 8.x
- Maven 3.8+（如果你要在服务器上编译）
- 可写的上传目录

当前默认配置见：

- `src/main/resources/application.yml`

默认配置重点如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/codexzy?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 123456

file:
  upload-dir: uploads

server:
  port: 8080
```

上线前你至少要改这几项：

- 数据库用户名
- 数据库密码
- 上传目录改成服务器绝对路径
- 必要时修改端口

## 2. 你现在这种情况怎么处理

如果你的服务器现在：

- 没有 MySQL
- 没有 JDK 17
- 不确定有没有 Maven

那就按这条顺序来：

1. 先装基础工具
2. 再装 Docker
3. 用 Docker 启动 MySQL 8
4. 安装 JDK 17
5. 决定要不要装 Maven
6. 拉项目代码
7. 改配置
8. 建表
9. 打包并启动
10. 验证页面和数据库

## 2.1 CentOS 7.6 专用说明

如果你的服务器是 `CentOS Linux release 7.6.x`，这里有几个现实问题要先知道：

1. CentOS Linux 7 已经在 **2024年6月30日** 结束生命周期。
2. 很多旧镜像源已经失效，所以你会遇到 `yum` 下载失败、源 404、包找不到的问题。
3. CentOS 7 正常用的是 `yum`，不是 `apt`，通常也不是 `dnf`。
4. JDK 17 在 CentOS 7 默认老源里不一定好装，最稳妥的方式是接 Adoptium 的 RPM 源来装。
5. Docker 也不建议再走太老的仓库说明，直接用官方安装脚本更省事。

如果你只想先把项目跑起来，建议你按这条最短路径：

1. 先确认 `yum` 能不能用
2. 如果 `yum` 源坏了，先切到 `vault`
3. 用 `yum` 装基础工具
4. 用 Adoptium 装 JDK 17
5. 用 Docker 官方脚本装 Docker
6. 用 Docker 跑 MySQL 8
7. 改项目配置并启动

先检查：

```bash
cat /etc/redhat-release
which yum
```

如果 `yum` 本身都没有，再往下贴结果给我，不要继续盲装。


```bash
mkdir -p /opt/codexzy
```

## 4. 安装 Docker

如果服务器还没有 Docker，推荐直接安装。

### 4.1 CentOS 7.6 先检查 yum 源

CentOS 7.6 很容易卡在这里，因为官方生命周期已经结束，旧源可能失效。

先试：

```bash
yum makecache
```

如果这里报错，先把基础源切到 `vault`：

```bash
sed -i 's|^mirrorlist=|#mirrorlist=|g' /etc/yum.repos.d/CentOS-Base.repo
sed -i 's|^#baseurl=http://mirror.centos.org/centos/$releasever|baseurl=http://vault.centos.org/7.9.2009|g' /etc/yum.repos.d/CentOS-Base.repo
yum clean all
yum makecache
```

说明：

- 这里虽然你的系统是 7.6，但实际装包建议切到 `7.9.2009` 的 vault 仓库继续用
- 这是为了让老系统还能继续拿到 CentOS 7 最后一版的可用包

### 4.2 通用快速安装方式

```bash
curl -fsSL https://get.docker.com | bash
```

安装完成后启动并设置开机自启：

```bash
systemctl enable docker
systemctl start docker
systemctl status docker
```

确认版本：

```bash
docker --version
```

如果你还想用 `docker compose`，再检查：

```bash
docker compose version
```

## 5. 安装 JDK 17

这个项目必须用 Java 17+，不能用 Java 8。

### 5.1 Ubuntu / Debian 安装 JDK 17

```bash
apt update
apt install -y openjdk-17-jdk
```

### 5.2 CentOS 7 安装 JDK 17

CentOS 7 默认老源里不一定能顺利装到可用的 OpenJDK 17，建议直接接 Adoptium 源。

先创建 repo 文件：

```bash
cat > /etc/yum.repos.d/adoptium.repo <<'EOF'
[Adoptium]
name=Adoptium
baseurl=https://packages.adoptium.net/artifactory/rpm/centos/$releasever/$basearch
enabled=1
gpgcheck=1
gpgkey=https://packages.adoptium.net/artifactory/api/gpg/key/public
EOF
```

然后执行：

```bash
yum clean all
yum makecache
yum install -y temurin-17-jdk
```

### 5.3 Rocky / AlmaLinux / CentOS Stream 安装 JDK 17

```bash
dnf install -y java-17-openjdk java-17-openjdk-devel
```

### 5.4 验证 Java

```bash
java -version
javac -version
```

你应该看到类似：

```text
openjdk version "17.x.x"
```

如果看到的是 `1.8`，说明还没切对。

### 5.5 配置 JAVA_HOME

先找 Java 安装路径：

```bash
readlink -f $(which java)
```

常见结果类似：

```text
/usr/lib/jvm/java-17-openjdk-amd64/bin/java
```

那 `JAVA_HOME` 就是：

```text
/usr/lib/jvm/java-17-openjdk-amd64
```

把它写到环境变量：

```bash
cat >> /etc/profile <<'EOF'
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
EOF
source /etc/profile
```

再次确认：

```bash
java -version
echo $JAVA_HOME
```

## 6. Maven 要不要装

这里有两种方式。

### 方式 A：在服务器上编译

这种方式需要 Maven。

Ubuntu / Debian：

```bash
apt install -y maven
```

CentOS 7：

```bash
yum install -y maven
```

Rocky / AlmaLinux / CentOS Stream：

```bash
dnf install -y maven
```

检查：

```bash
mvn -v
```

这里最关键的是 `mvn -v` 里显示的 Java 版本必须是 17+。

### 方式 B：本地打包，服务器只运行 Jar

如果你不想在服务器装 Maven，可以在你自己电脑上执行：

```bash
mvn clean package -DskipTests
```

然后把下面这个文件传到服务器：

```text
target/codexzy-1.0.0.jar
```

这种方式服务器只需要：

- JDK 17
- MySQL

不一定要 Maven。

如果你是 CentOS 7.6，而且 `yum install maven` 也因为源问题折腾很久，那就直接改走“方式 B”：

1. 在你自己电脑本地打包
2. 把 Jar 传到服务器
3. 服务器只保留 JDK 17 和 MySQL

这样能少踩很多老系统环境坑。

## 7. 用 Docker 安装并启动 MySQL 8

### 7.1 拉取镜像

```bash
docker pull mysql:8.0
```

### 7.2 启动 MySQL 容器

```bash
docker run -d \
  --name codexzy-mysql \
  --restart unless-stopped \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456XZY \
  -e TZ=Asia/Shanghai \
  -v /data/codexzy/mysql:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

说明：

- `MYSQL_ROOT_PASSWORD=123456` 只是演示值，正式环境建议你换成强密码
- `/data/codexzy/mysql` 是 MySQL 数据持久化目录
- `--restart unless-stopped` 表示重启服务器后容器会自动起来

### 7.3 查看容器状态

```bash
docker ps
```

你应该能看到：

- 容器名：`codexzy-mysql`
- 端口：`3306->3306`

### 7.4 查看 MySQL 启动日志

```bash
docker logs -f codexzy-mysql
```

看到类似 `ready for connections` 基本就说明好了。

## 8. 初始化数据库

### 8.1 进入 MySQL

```bash
docker exec -it codexzy-mysql mysql -uroot -p123456
```

### 8.2 创建数据库

进入 MySQL 后执行：

```sql
CREATE DATABASE IF NOT EXISTS codexzy
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 8.3 推荐创建独立业务账号

不要长期直接用 root 跑应用，建议创建业务用户：

```sql
CREATE USER IF NOT EXISTS 'codexzy'@'%' IDENTIFIED BY '请替换成强密码';
GRANT ALL PRIVILEGES ON codexzy.* TO 'codexzy'@'%';
FLUSH PRIVILEGES;
```

如果你只允许本机访问，也可以用：

```sql
CREATE USER IF NOT EXISTS 'codexzy'@'localhost' IDENTIFIED BY '请替换成强密码';
GRANT ALL PRIVILEGES ON codexzy.* TO 'codexzy'@'localhost';
FLUSH PRIVILEGES;
```

### 8.4 查看数据库是否创建成功

```sql
SHOW DATABASES;
```

确认能看到：

```text
codexzy
```

## 10. 修改项目配置

需要改的文件：

- `src/main/resources/application.yml`

### 10.1 修改数据库配置

如果你用了上面创建的业务账号，建议改成：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/codexzy?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: codexzy
    password: 你刚设置的强密码
```

### 10.2 修改上传目录

默认配置里是：

```yaml
file:
  upload-dir: uploads
```

上线建议改成绝对路径：

```yaml
file:
  upload-dir: /data/codexzy/uploads
```

### 10.3 检查端口

默认是：

```yaml
server:
  port: 8080
```

如果你的服务器 8080 已经被其他服务占用，可以改成 8081、8090 等。

### 10.4 生产环境建议关闭 MyBatis SQL 控制台日志

当前配置里有：

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

正式环境建议去掉这一行，避免控制台日志过多。

## 12. 数据表怎么初始化

当前项目已经配置了：

```yaml
spring:
  sql:
    init:
      mode: always
```

并且项目自带建表脚本：

- `src/main/resources/schema.sql`

也就是说：

- 应用首次启动时，会自动执行 `schema.sql`
- 表不存在就会创建

脚本里会创建这些表：

- `user`
- `check_in`
- `check_in_file`
- `memo_category`
- `memo_file`

### 12.1 自动建表方式

最简单，直接启动项目，让 Spring Boot 自己执行。

### 12.2 手动导入方式

如果你想手动建表，可以执行：

```bash
docker exec -i codexzy-mysql mysql -uroot -p123456 codexzy < /opt/codexzy/src/main/resources/schema.sql
```

然后检查：

```bash
docker exec -it codexzy-mysql mysql -uroot -p123456 -e "USE codexzy; SHOW TABLES;"
```

## 17. 建议加 systemd，避免服务器重启后服务没起来

新建文件：

```bash
vim /etc/systemd/system/codexzy.service
```

内容如下：

```ini
[Unit]
Description=codexzy Spring Boot Application
After=network.target docker.service

[Service]
Type=simple
WorkingDirectory=/opt/codexzy
ExecStart=/usr/bin/java -jar /opt/codexzy/target/codexzy-1.0.0.jar
Restart=always
RestartSec=5
User=root
Environment=JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64

[Install]
WantedBy=multi-user.target
```

然后执行：

```bash
systemctl daemon-reload
systemctl enable codexzy
systemctl start codexzy
systemctl status codexzy
```

查看日志：

```bash
journalctl -u codexzy -f
```

如果你的 `java` 不在 `/usr/bin/java`，先用下面命令确认：

```bash
which java
```

然后把 `ExecStart` 改成实际路径。

## 18. 防火墙和端口

至少要关注这些端口：

- `8080`：Spring Boot 对外访问端口
- `3306`：MySQL 端口

建议：

- `8080` 可以按需开放
- `3306` 尽量不要直接对公网开放

Ubuntu 如果用 `ufw`：

```bash
ufw allow 8080/tcp
ufw status
```

CentOS / Rocky 如果用 `firewalld`：

```bash
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload
firewall-cmd --list-ports
```

## 19. 推荐正式环境再补一个 Nginx

如果以后你要用域名和 HTTPS，建议前面再加一层 Nginx：

- `80/443` 给 Nginx
- `8080` 只让应用自己监听
- Nginx 反向代理到 `127.0.0.1:8080`

简单示例：

```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 20. 常见问题

### 20.1 `mvn -v` 还是显示 Java 8

原因通常是：

- `JAVA_HOME` 还没切到 17
- shell 没重新加载环境变量
- Maven 启动时拿到的仍然是旧 Java

处理顺序：

1. 重新确认 `java -version`
2. 重新确认 `echo $JAVA_HOME`
3. 执行 `source /etc/profile`
4. 再执行 `mvn -v`

### 20.2 MySQL 容器启动了，但应用连不上

检查：

```bash
docker ps
docker logs codexzy-mysql
```

再检查：

- 数据库名是不是 `codexzy`
- 用户名密码是不是和 `application.yml` 一致
- 端口是不是 `3306`
- `url` 里是不是写成了 `127.0.0.1:3306`

### 20.3 页面提交 403

这个项目表单里已经带了 CSRF 字段。  
如果你后面自己加新表单，也必须带上：

```html
<input type="hidden" th:name="${_csrf.parameterName}" th:value="${_csrf.token}">
```

### 20.4 上传文件失败

优先检查：

- `file.upload-dir` 是不是绝对路径
- 目录是不是存在
- 目录权限是不是可写

检查命令：

```bash
ls -ld /data/codexzy/uploads
```

### 20.5 启动时报端口占用

检查 8080：

```bash
ss -lntp | grep 8080
```

如果占用了：

- 改 `application.yml` 里的 `server.port`
- 或停掉旧进程

### 20.6 启动后数据库表没建出来

先检查应用日志里有没有 `schema.sql` 执行失败。  
如果不想查日志，直接手动执行：

```bash
docker exec -i codexzy-mysql mysql -uroot -p123456 codexzy < /opt/codexzy/src/main/resources/schema.sql
```
