# codexzy 上线部署指南

本文档按“从一台新机器开始部署”的顺序整理，适合本地联调、测试环境部署和简单生产环境上线。

## 1. 部署目标

当前项目是一个基于以下技术栈的单体应用：

- Java 17+
- Spring Boot 3.2.x
- Maven
- MySQL 8.x
- Thymeleaf
- Spring Security
- MyBatis-Plus

当前版本已实现：

- 用户注册
- 用户登录
- 首页统计面板
- 个人信息修改
- 修改密码
- 打卡列表
- 新建打卡
- 删除打卡

当前版本未完成：

- 打卡附件上传
- 打卡日历视图
- 备忘录真实业务逻辑
- 更完整的生产级安全配置

## 2. 部署方式概览

推荐按下面的顺序执行：

1. 安装 Docker Desktop
2. 使用 Docker 拉起 MySQL 8
3. 创建应用数据库
4. 准备 JDK 17 或更高版本
5. 准备 Maven
6. 拉取项目代码
7. 修改项目配置
8. 编译项目
9. 启动 Spring Boot 应用
10. 访问页面并验证

## 3. 机器准备

### 3.1 安装 Docker Desktop

如果你的机器还没有 Docker，先安装 Docker Desktop。

Windows 常规要求：

- Windows 10/11
- 已启用虚拟化
- 已启用 WSL2 或 Hyper-V

安装完成后，先确认 Docker 可用：

```bash
docker --version
docker compose version
```

### 3.2 安装 JDK 17 或更高版本

本项目必须使用 Java 17 或更高版本。

先检查当前 Java：

```bash
java -version
javac -version
mvn -v
```

必须确认：

- `java -version` 至少为 17
- `javac -version` 至少为 17
- `mvn -v` 显示的 Java 版本也至少为 17

如果 `mvn -v` 仍然显示 Java 8，说明你的 `JAVA_HOME` 没切对。

### 3.3 安装 Maven

如果机器还没有 Maven，可以先安装 Maven 3.8+。

安装后检查：

```bash
mvn -v
```

## 4. 使用 Docker 启动 MySQL

### 4.1 拉取 MySQL 镜像

```bash
docker pull mysql:8.0
```

### 4.2 创建数据目录

建议先在宿主机准备一个目录，方便持久化 MySQL 数据。

例如在 Windows：

```powershell
mkdir C:\docker-data\codexzy-mysql
```

### 4.3 启动 MySQL 容器

执行：

```bash
docker run -d ^
  --name codexzy-mysql ^
  -p 3306:3306 ^
  -e MYSQL_ROOT_PASSWORD=root ^
  -e TZ=Asia/Shanghai ^
  -v C:\docker-data\codexzy-mysql:/var/lib/mysql ^
  mysql:8.0 ^
  --character-set-server=utf8mb4 ^
  --collation-server=utf8mb4_unicode_ci
```

如果你是 Linux 或 macOS，可把挂载目录改成对应路径，例如：

```bash
docker run -d \
  --name codexzy-mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root \
  -e TZ=Asia/Shanghai \
  -v /data/codexzy-mysql:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

### 4.4 检查容器状态

```bash
docker ps
```

看到 `codexzy-mysql` 正常运行即可。

### 4.5 查看 MySQL 日志

如果容器启动失败，先看日志：

```bash
docker logs codexzy-mysql
```

## 5. 初始化数据库

### 5.1 进入 MySQL 容器

```bash
docker exec -it codexzy-mysql mysql -uroot -proot
```

### 5.2 创建数据库

进入 MySQL 命令行后执行：

```sql
CREATE DATABASE IF NOT EXISTS codexzy
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;
```

然后确认：

```sql
SHOW DATABASES;
```

看到 `codexzy` 即可。

### 5.3 表初始化说明

当前项目配置了：

- `spring.sql.init.mode=always`

并且项目里已经提供了建表脚本：

- [schema.sql](/C:/Users/admin/Desktop/git-java/codexzy/src/main/resources/schema.sql)

因此应用启动时会自动执行该脚本，自动创建以下表：

- `user`
- `check_in`
- `check_in_file`
- `memo_category`
- `memo_file`

### 5.4 手动执行建表脚本

如果你不想依赖 Spring Boot 自动初始化，也可以手动导入：

```bash
docker exec -i codexzy-mysql mysql -uroot -proot codexzy < src/main/resources/schema.sql
```

执行后可检查：

```bash
docker exec -it codexzy-mysql mysql -uroot -proot -e "USE codexzy; SHOW TABLES;"
```

## 7. 修改应用配置

当前配置文件在：

- [application.yml](/C:/Users/admin/Desktop/git-java/codexzy/src/main/resources/application.yml)

默认数据库配置是：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/codexzy?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root
```

如果你修改了 MySQL 端口、用户名或密码，需要同步调整这里。

### 7.1 上传目录

当前配置：

```yaml
file:
  upload-dir: uploads
```

建议上线时把它改成绝对路径，例如：

```yaml
file:
  upload-dir: /data/codexzy/uploads
```

Windows 也可以使用：

```yaml
file:
  upload-dir: C:/codexzy/uploads
```

### 7.2 端口

当前项目默认端口：

```yaml
server:
  port: 8080
```

如果机器上 8080 被占用，可以改成其他端口，例如：

```yaml
server:
  port: 8081
```

## 8. 编译项目

### 8.1 先确认 Maven 使用的是正确 JDK

```bash
mvn -v
```

你必须看到类似：

```text
Java version: 17
```

或者更高版本。

### 8.2 编译

```bash
mvn clean compile
```

如果只想快速验证主代码，不跑测试：

```bash
mvn -DskipTests compile
```

### 8.3 打包

```bash
mvn clean package -DskipTests
```

打包成功后，产物一般在：

```text
target/codexzy-1.0.0.jar
```

### 9.1 开发环境直接启动

```bash
mvn spring-boot:run
```

### 9.2 使用 Jar 包启动

```bash
java -jar target/codexzy-1.0.0.jar
```

### 9.3 指定配置启动

如果你后面要拆分测试环境和生产环境配置，可以这样启动：


## 11. 生产环境建议

当前项目可以上线跑通，但如果要真正长期稳定运行，建议补这些项：

### 11.1 修改数据库密码

不要继续使用：

- root/root

建议创建单独业务账号，例如：

```sql
CREATE USER 'codexzy'@'%' IDENTIFIED BY '请替换成强密码';
GRANT ALL PRIVILEGES ON codexzy.* TO 'codexzy'@'%';
FLUSH PRIVILEGES;
```

然后把 `application.yml` 改成新账号。

### 11.2 关闭开发日志

当前 MyBatis-Plus 开启了 SQL 控制台日志：

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

正式环境建议去掉，避免日志过多和敏感信息暴露。

### 11.3 上传目录放到独立磁盘

不要把上传文件直接跟代码目录混在一起。

建议：

- Linux: `/data/codexzy/uploads`
- Windows: `D:/codexzy/uploads`

### 11.4 反向代理

如果要对外提供访问，建议前面加 Nginx 或网关，处理：

- 域名
- HTTPS
- 反向代理
- 静态资源缓存

### 11.5 日志落盘

当前项目没有单独日志配置。

正式环境建议增加：

- `logback-spring.xml`
- 日志滚动策略
- 错误日志单独文件

### 11.6 数据备份

MySQL 容器虽然挂载了数据目录，但仍建议定期备份数据库：

```bash
docker exec codexzy-mysql mysqldump -uroot -proot codexzy > codexzy-backup.sql
```

## 12. 常见问题

### 12.1 `mvn -v` 显示 Java 8

原因：

- `JAVA_HOME` 还是旧 JDK
- Maven 启动时拿到的是旧 Java

处理：

1. 安装 JDK 17+
2. 修改 `JAVA_HOME`
3. 重新打开终端
4. 再执行 `mvn -v`

### 12.2 应用启动失败，提示数据库连接不上

检查：

1. `docker ps` 中 MySQL 是否在运行
2. 3306 端口是否映射成功
3. `application.yml` 的用户名密码是否正确
4. `codexzy` 数据库是否存在

### 12.3 页面提交 403

当前项目已经在表单里加入了 CSRF 字段。

如果你后续自己新增表单，也必须带上：

```html
<input type="hidden" th:name="${_csrf.parameterName}" th:value="${_csrf.token}">
```

### 12.4 页面能打开但样式没加载

检查：

- `src/main/resources/static/css/app.css` 是否存在
- 页面里是否使用 `th:href="@{/css/app.css}"`
- Spring Boot 是否正常启动

### 12.5 启动时报表已存在

当前建表脚本使用了：

```sql
CREATE TABLE IF NOT EXISTS
```

一般不会因重复建表报错。

如果你手动改过表结构，建议先备份再调整。
