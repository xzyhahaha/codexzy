# 开发记录

## 说明

用于记录本项目每次功能开发、修复、配置调整和文档更新，方便后续排查问题和持续维护。

## 维护规则

- 每次完成代码、配置、数据库、模板或接口变更后，先更新本文档，再进行提交或交付。
- 如果本次改动影响接口、页面流程、字段含义、数据库结构或业务规则，也要同步更新相关设计、规则或说明文档。
- 记录按时间倒序追加，保持简洁、客观、可追溯。
- 只记录事实，不写主观评价。
